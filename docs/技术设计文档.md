### 1.2 架构设计原则
1. **前后端分离**：前端独立部署，通过API与后端通信
2. **微服务思想**：模块化设计，服务间松耦合
3. **安全性优先**：API密钥隔离，输入验证，数据加密
4. **可扩展性**：支持水平扩展，模块可独立升级
5. **监控可观测**：完善的日志、监控和告警机制

## 3. 数据库设计

### 3.1 核心数据表

#### 表：users (用户表)
```sql
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    email VARCHAR(255) UNIQUE NOT NULL,
    company_name VARCHAR(255),
    industry VARCHAR(100),
    business_description TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    last_login_at TIMESTAMP
);
```

#### 表：hotspots (热点表)
```sql
CREATE TABLE hotspots (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    title VARCHAR(500) NOT NULL,
    summary TEXT,
    content_url VARCHAR(1000),
    source_type VARCHAR(50), -- 'news', 'tech_blog', 'social_media'
    source_name VARCHAR(200),
    publish_date TIMESTAMP,
    collected_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    raw_content TEXT,
    processed_content JSONB, -- 存储解析后的结构化内容
    tags TEXT[], -- 标签数组
    metadata JSONB -- 额外元数据
);
```

#### 表：hotspot_analyses (热点分析表)
```sql
CREATE TABLE hotspot_analyses (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    hotspot_id UUID REFERENCES hotspots(id) ON DELETE CASCADE,
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    relevance_score INTEGER, -- 0-100相关度分数
    importance_level VARCHAR(20), -- 'emergency', 'high', 'medium', 'low', 'watch'
    business_impact TEXT, -- 业务影响分析
    importance_reason TEXT, -- 重要性解释
    action_suggestions TEXT, -- 行动建议
    technical_details TEXT, -- 技术细节
    analyzed_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    analysis_metadata JSONB -- 分析过程元数据
);
```

#### 表：user_settings (用户设置表)
```sql
CREATE TABLE user_settings (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    update_schedule VARCHAR(50) DEFAULT 'daily_2am', -- 更新计划
    notification_preferences JSONB, -- 通知偏好
    display_preferences JSONB, -- 显示偏好
    filter_preferences JSONB, -- 筛选偏好
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 表：api_usage (API使用记录表)
```sql
CREATE TABLE api_usage (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) ON DELETE SET NULL,
    endpoint VARCHAR(200),
    model_used VARCHAR(100),
    tokens_used INTEGER,
    cost DECIMAL(10, 6), -- API调用成本
    response_time_ms INTEGER,
    success BOOLEAN,
    error_message TEXT,
    requested_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 3.2 数据关系图
```
users
  ├── hotspot_analyses (一对多)
  └── user_settings (一对一)

hotspots
  └── hotspot_analyses (一对多)

hotspot_analyses (连接 users 和 hotspots)
```

### 6.2 输入验证
```python
# FastAPI依赖注入验证
from pydantic import BaseModel, validator

class BusinessDescription(BaseModel):
    description: str
    
    @validator('description')
    def validate_description_length(cls, v):
        if len(v) < 10:
            raise ValueError('业务描述至少10个字符')
        if len(v) > 5000:
            raise ValueError('业务描述最多5000个字符')
        return v
```

### 6.3 访问控制

- **CORS配置**：严格限制允许的源
- **SQL注入防护**：使用ORM参数化查询
- **XSS防护**：输出编码，Content Security Policy

## 7. 部署架构

### 7.1 开发环境部署
```yaml
# docker-compose.dev.yml
version: '3.8'
services:
  postgres:
    image: postgres:15
    environment:
      POSTGRES_DB: hotspot_analysis
      POSTGRES_USER: admin
      POSTGRES_PASSWORD: dev_password
    volumes:
      - postgres_data:/var/lib/postgresql/data
  
  redis:
    image: redis:7-alpine
  
  backend:
    build: ./backend
    environment:
      DATABASE_URL: postgresql://admin:dev_password@postgres/hotspot_analysis
      REDIS_URL: redis://redis:6379/0
      LLM_API_KEY: ${LLM_API_KEY}
    ports:
      - "8000:8000"
    depends_on:
      - postgres
      - redis
  
  celery-worker:
    build: ./backend
    command: celery -A app.celery_app worker --loglevel=info
    environment: # 同backend环境变量
    depends_on:
      - postgres
      - redis
  
  celery-beat:
    build: ./backend
    command: celery -A app.celery_app beat --loglevel=info
    environment: # 同backend环境变量
    depends_on:
      - postgres
      - redis
  
  frontend:
    build: ./frontend
    ports:
      - "3000:3000"
    environment:
      VITE_API_BASE_URL: http://localhost:8000/api/v1

volumes:
  postgres_data:
```
