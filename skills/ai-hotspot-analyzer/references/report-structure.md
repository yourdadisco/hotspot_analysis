# AI热点分析报告结构模板

## 报告结构规范

所有AI热点分析报告应遵循以下结构，确保信息完整性和可读性。

---

## 1. 报告头部信息
```
# AI行业热点分析报告
**分析时间**: 2026-04-21
**时间范围**: 最近24-72小时热点
**业务背景**: [用户提供的业务描述]
**报告生成**: AI业务趋势分析器
```

## 2. 执行摘要 (限300字)
- 本次分析的核心发现
- 热点总数及分布情况
- 关键趋势概述
- 对业务的总体影响评估

## 3. 业务背景确认
```
### 3.1 用户业务描述
[在此记录用户提供的业务描述，确保理解准确]

### 3.2 分析重点
基于业务描述，本次分析重点关注：
1. [技术领域1]
2. [技术领域2]
3. [应用场景1]
...
```

## 4. 热点排行榜 (核心部分)

### 高优先级热点 (得分≥80)

#### 热点标题
**相关度得分**: XX/100  
**重要性分级**: 高优先级  
**简要描述**: [2-3句话描述热点内容]  
**关键理由分析**:
- **技术相关性**: [分析技术匹配度]
- **业务影响**: [分析对业务的具体影响]
- **时效性**: [分析技术发展阶段]
- **综合评估**: [总结为什么重要]

**建议行动项**:
1. [具体建议1]
2. [具体建议2]

---

### 中优先级热点 (得分60-79)

#### 热点标题
**相关度得分**: XX/100  
**重要性分级**: 中优先级  
**简要描述**: [1-2句话描述]  
**关键理由分析**: [简要分析]

**建议**: [跟踪建议]

---

### 低优先级热点 (得分<60)

#### 热点标题
**相关度得分**: XX/100  
**重要性分级**: 低优先级  
**简要描述**: [1句话描述]  
**备注**: [简要说明相关性较弱的原因]

## 5. 深度分析 (可选)

对特别重要的高优先级热点进行深入分析：

### 深度分析：[热点标题]
**技术原理**: [技术细节说明]  
**行业应用**: [当前应用案例]  
**发展趋势**: [未来1-2年发展预测]  
**竞争格局**: [主要参与者和竞争态势]  
**风险与挑战**: [实施风险和挑战]

## 6. 综合建议与行动计划

### 6.1 短期行动 (1-3个月)
1. **立即行动项**: [需要立即开始的工作]
2. **研究评估**: [需要进一步研究的技术]

### 6.2 中期规划 (3-12个月)
1. **技术跟踪**: [需要持续关注的技术趋势]
2. **能力建设**: [团队需要培养的技能]

### 6.3 长期战略 (1年以上)
1. **战略布局**: [基于趋势的战略调整建议]
2. **创新方向**: [可能的创新机会]

## 7. 附录

### 7.1 分析方法说明
- **搜索关键词**: [本次使用的搜索关键词]
- **信息源**: [主要信息来源]
- **分析框架**: [相关度分析的具体维度]
- **分级标准**: [重要性分级的具体标准]

### 7.2 免责声明
- 本报告基于公开信息分析，仅供参考
- 热点信息具有时效性，建议定期更新分析
- 具体业务决策需结合实际情况

---

## 格式要求

1. **标题层级**: 使用规范的Markdown标题层级 (#, ##, ###)
2. **重点突出**: 使用**粗体**强调关键信息
3. **列表清晰**: 使用有序和无序列表提高可读性
4. **表格使用**: 复杂数据可使用表格呈现
5. **字数控制**: 报告总字数控制在1500-3000字

## 质量检查清单

- [ ] 报告头部信息完整
- [ ] 执行业务背景准确记录
- [ ] 热点按优先级排序
- [ ] 每个热点都有相关度得分和分级
- [ ] 分析理由充分具体
- [ ] 建议行动项切实可行
- [ ] 格式规范，易于阅读
- [ ] 严格过滤八卦信息
